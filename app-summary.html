Styles By Tiwa App Summary

One-page repo-based overview of the static storefront and admin tooling.

What It Is

A static fashion storefront for Styles By Tiwa with a landing page, a shop page, and a browser-based admin area. The customer-facing pages are plain HTML/CSS/JS, while product data, image storage, and admin authentication are handled through Supabase.

Who It’s For

Primary persona: a small fashion brand owner in Lagos who wants to showcase collections online, take orders through WhatsApp, and manage products without a custom backend app.

What It Does

  • Presents a branded homepage with hero, about, featured collection, and contact sections.
  • Shows a shop grid populated from a Supabase products table.
  • Lets shoppers filter products by category.
  • Opens product details in a modal with title, image, price, and CTA.
  • Sends purchase intent to WhatsApp with prefilled order text.
  • Supports admin login with Supabase Auth and a single allowed email check.
  • Lets the admin add, edit, delete, and image-upload products via Supabase Storage.

How It Works

UI components: index.html (landing), shop.html (catalog), login.html (admin auth), tiwa-admin-dashboard.html (product management), all styled by style.css.

Client logic: script.js toggles mobile nav; shop.js loads and filters products; admin.js protects the admin page and performs CRUD.

Services: Supabase JS CDN is used for auth, database reads/writes against products, and uploads to the product-images storage bucket. Google Maps and social/media links are embedded directly in the pages.

Data flow: Browser page -> JS creates Supabase client -> shop reads products and renders cards/modal -> admin authenticates, uploads images, then inserts/updates/deletes product records -> shopper orders continue in WhatsApp. Server code/API layer: Not found in repo.

How To Run

  1. Open index.html in a browser for the landing page.
  2. Open shop.html for the storefront catalog; internet access is needed for Supabase/CDN assets.
  3. Open login.html to access admin features, then continue to tiwa-admin-dashboard.html after sign-in.
  4. Supabase schema setup, local dev server, package manager commands, and deployment steps: Not found in repo.
